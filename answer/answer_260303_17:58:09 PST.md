## Recommended Prompt Template

```
I'm building a [workload description] that uses the following Azure resources:
- [Resource 1] (in [landing zone])
- [Resource 2] (in [landing zone])
- [Resource 3] (shared/platform)
...

Give me the least-privileged RBAC roles to create each resource, their cross-resource 
dependencies, and the roles needed to access those dependencies. 
Save the output to `rbac-summary-[workload-name].md`.
```

---

## What Makes a Prompt Comprehensive

### 1. Name every resource explicitly
Don't say "an AI app" — list the actual services:
> ❌ "I have an AI workload"
> ✅ "I have: Azure AI Foundry, Azure OpenAI, Azure AI Search, Azure Storage Account (for blobs), and Azure Key Vault"

### 2. Specify the sub-resource type when relevant
> ✅ "Azure Storage Account — specifically Blob storage for model artifacts"
> ✅ "Azure Key Vault — secrets only, not certificates"

### 3. Include shared/platform resources
> ✅ "Also include: Log Analytics Workspace (platform-owned), Azure Container Registry (shared)"

### 4. Specify the identity type
> ✅ "Roles for: (a) the human operator deploying the solution, (b) the app's Managed Identity at runtime"

### 5. Ask for the output file upfront
> ✅ "Save the full RBAC summary to `rbac-[solution-name].md`"

## Example Prompt — AI Solution Spanning Multiple Zones

```
I'm building a RAG (Retrieval-Augmented Generation) solution with these resources:
- Azure AI Foundry (AI Landing Zone)
- Azure OpenAI (AI Landing Zone)
- Azure AI Search (AI Landing Zone)
- Azure Storage Account — Blob only (Workload Landing Zone)
- Azure Key Vault — secrets only (Workload Landing Zone)
- Azure SQL Database — read-only by the app (Workload Landing Zone)

Give me:
1. The least-privileged role to CREATE each resource (human operator / deployment identity)
2. The cross-resource dependencies and roles needed at runtime (Managed Identity)

Save the output to `rbac-summary-rag-solution.md`.
```
