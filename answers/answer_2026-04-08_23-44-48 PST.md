## Original Prompt

I have an application hosted in a Container App and it need to access Azure AI Search. I need to find what role is needed for the app's managed identity and I need to know if I should be assigning the managed identity of the container app or the container app environment.

---

## Which Managed Identity to Use

**Assign the role to the Container App's managed identity**, not the Container Apps Environment's.

- The **Container App** (`Microsoft.App/containerApps`) supports both system-assigned and user-assigned managed identities. This is the identity your application code uses when calling Azure services at runtime (e.g., pulling images from ACR, reading secrets from Key Vault, querying AI Search).
- The **Container Apps Environment** (`Microsoft.App/managedEnvironments`) also has a managed identity, but it is used for environment-level concerns only — TLS certificates from Key Vault for custom domains and Dapr component authentication. It is **not** the identity your application code runs under.

## Role Required on Azure AI Search

| Operation | Scope | Least-Privileged Role | Notes |
|---|---|---|---|
| Query an index (search/suggest/autocomplete) | Search Service | `Search Index Data Reader` | Minimum role for application search queries — covers full-text, vector, and semantic search. |
| Query + push documents to index | Search Service | `Search Index Data Contributor` | Only if the app also uploads/updates documents in the index. |

### Data Plane Role Summary

| Role | Query Index | Upload Documents | Create/Delete Index | Manage Service |
|---|---|---|---|---|
| `Search Index Data Reader` | ✅ | ❌ | ❌ | ❌ |
| `Search Index Data Contributor` | ✅ | ✅ | ✅ | ❌ |

## Recommendation

- If your Container App **only queries** the search index (typical for a web app or API), assign **`Search Index Data Reader`** to the Container App's managed identity, scoped to the AI Search service resource.
- If your Container App also **pushes documents** into the index, assign **`Search Index Data Contributor`** instead.
- Disable API key authentication on the AI Search service (`disableLocalAuth: true`) and use Entra ID token auth exclusively.

## Notes

- Managed Identity is the preferred authentication pattern for Container Apps accessing Azure services.
- The Container Apps Environment's managed identity should **not** be used for application-level service access — it is scoped to environment infrastructure (Dapr, custom domain certs).
- Scope the role assignment to the specific AI Search service resource, not the resource group or subscription.
